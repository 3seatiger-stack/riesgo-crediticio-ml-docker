# Índice de Evidencias

Esta carpeta debe contener capturas reales del proceso.

## Estructura recomendada

```text
evidence/
├── docker_build/
│   ├── 01_docker_build_exitoso.png
│   └── 02_docker_images.png
├── local_execution/
│   ├── 01_contenedor_ejecucion.png
│   ├── 02_frontend.png
│   └── 03_resultado_prediccion.png
├── api_tests/
│   ├── 01_swagger_endpoints.png
│   ├── 02_predict_200.png
│   ├── 03_pytest_9_passed.png
│   └── 04_github_actions_green.png
└── cloud/
    ├── 01_cloud_run_servicio.png
    ├── 02_cloud_run_url_publica.png
    └── 03_app_publica.png
```

## Recomendación

Cada captura debe:

- mostrar fecha o contexto cuando sea posible;
- evitar información sensible;
- corresponder a una ejecución real;
- mantener resolución legible;
- utilizar nombres consistentes.

## Nota de integridad académica

No deben fabricarse ni editarse capturas para aparentar resultados no obtenidos.
